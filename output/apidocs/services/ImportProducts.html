ImportProducts

This service lets you create and update a set of products and their content. The set is products is hierarchical in nature. For example, if you have a shirt that comes in 3 colors and 5 sizes per color, you would normally define a root product that then contains the 3 color products and each color product would contain the 5 different sizes, which would be the SKU. How you define the hierarchy is up to you, it's also fine to have the 3 color products as root products and only have them contain the SKUs, or to forego a hierarchy entirely and just create the SKUs, but creating the root and color products is useful for search and frontend purposes. An example request of a hierarchy: { "ID": "special_shirt", "Name": "Special Shirt", "TaxCode": "High", "Variations": { "Property": "color_code", "LogicalLevel": "color", "Products": [ { "ID": "special_shirt_blue", "Name": "Special Shirt - Blue", "VariationValues": [ { "LanguageID": null, "Value": "BLUE" } ], "Variations": { "Property": "size_code", "LogicalLevel": "size", "Products": [ { "ID": "sku_special_shirt_blue_xl", "VariationValues": [ { "LanguageID": null, "Value": "XL" } ] } ] } } ] } }
